# Traceability Module - Golden Rules

1. Diagnosis is the semantic source.
2. Line-by-Line Classification is the primary extraction source.
3. Numbered SP is the final source of truth for line ranges.
4. Never invent semantics.
5. Never generate architecture.
6. Never generate code.
7. Never silently succeed with zero blocks.
8. Unknown values must be explicit, never guessed.
9. Every block must be line-anchored.
10. Every block must have semantic identity.
11. Enrichment is secondary; extraction is primary.
12. Determinism is mandatory.
13. Debuggability is mandatory during MVP.
14. Traceability is a bridge, not a design layer.
15. If extraction fails, fail explicitly.
